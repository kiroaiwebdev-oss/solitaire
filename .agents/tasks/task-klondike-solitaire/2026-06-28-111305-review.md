# Fix commit for 7 review issues (v2)

This commit addresses all seven concerns from the v1 review: DPR coordinate mismatch, per-frame felt texture cost, snap-back teleportation, renderer scale accumulation, game-state persistence on unload, spurious reveal-card bonus, and premature auto-complete trigger. The fixes are well-targeted — each modifies only the code paths responsible for the bug, and all 627 test assertions continue to pass.

**Watch for:** The `_executeDrop` path duplicates scoring logic that also exists in `Game.moveTableauToTableau` — two parallel code paths for the same operation with independent scoring constants invite future drift (likely). Dead `_returnCards` method remains after the snap-back refactor (confirmed).

**Verdict**: APPROVED

## High-level view

The DPR and scale-accumulation fixes are paired: `setTransform` in `render.js` makes CSS pixels the drawing coordinate system unconditionally (no stale transform on multi-monitor drag), and input now returns raw CSS-pixel offsets to match.

The save/restore system serializes all pile contents by `(suit, rank)` identity. Restoration creates a throwaway deal to populate `allCards`, then overwrites every pile by card lookup — a design that avoids storing Card object references but depends on 52-card uniqueness.

The snap-back refactor restructured pointer-up: it reads `drag.cards`/`drag.source` directly before deciding, calls `end()` only on valid drops, and calls `snapBack()` on invalid ones. The old `_returnCards` method survives as dead code.

## Dual scoring paths

`App._executeDrop` — the only code path triggered by drag-and-drop — reimplements removal, placement, and scoring by calling `tableau.removeSequence`, `foundation.placeCard`, and manually adjusting `game.score`. The `Game` class has parallel methods (`moveTableauToTableau`, etc.) that do the same with their own scoring. Both now correctly gate the reveal bonus on a face-down check, but with independent implementations:

```js
// In Game.moveTableauToTableau:
const willReveal = newTopIndex >= 0 && !col[newTopIndex].faceUp;
if (willReveal) { this.score += 5; }

// In App._executeDrop:
if (newTopIndex >= 0 && !col[newTopIndex].faceUp) { sourceTopWasFaceDown = true; }
if (source.type === 'tableau' && sourceTopWasFaceDown) { this.game.score += SCORING.REVEAL_CARD; }
```

The `Game` methods use hardcoded `5` while `_executeDrop` uses `SCORING.REVEAL_CARD`. If that config value changes, the auto-complete path (which calls `Game` methods) and the drag path (which uses `_executeDrop`) would award different points for the same action.

## Orphaned `_returnCards`

```js
_returnCards(source, cards) {
  for (const card of cards) {
    card.dragging = false;

```

Nothing calls this after the snap-back fix. Dead code that implies an alternate return path exists.
