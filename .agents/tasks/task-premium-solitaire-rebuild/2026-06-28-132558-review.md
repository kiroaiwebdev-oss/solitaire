# Premium Solitaire Rebuild (v2)

Six of the seven v1 issues have been correctly addressed: the event name mismatch (`'keyboard'` → `'shortcut'`) is fixed, double-tap now resolves coordinates to a Card object via `_findCardAt()`, screen state is pushed through `_syncScreenState()`, difficulty and scoring mode flow through to the Game constructor, the undo double-penalty is removed, and the stats chart uses deterministic bar heights. One critical issue remains: the keyboard handler still parses raw DOM `KeyboardEvent` properties (`e.key`, `e.ctrlKey`) instead of the `{ action }` object that Input actually emits. All shortcuts are still dead at runtime.

**Watch for:** Keyboard shortcut handler parses wrong event shape — `e.key` is `undefined` when Input emits `{ action: 'undo' }`, so no conditions match (confirmed). Daily challenge calendar always empty due to missing arguments to `getCalendarMonth()` (confirmed).

**Verdict**: NEEDS_CHANGES

## High-level view

The keyboard event name was fixed, but the handler body was never rewritten to consume the `{ action }` payload that Input emits. The handler checks `e.key`, `e.ctrlKey`, `e.shiftKey` — all `undefined` on the `{ action: '...' }` object — so every keyboard shortcut still silently fails. This is the only blocking concern.

The `_syncScreenState` bridge correctly pushes achievements, progression, and daily data to screens after load and save, but calls `getCalendarMonth()` without the required `(year, month)` arguments. The string comparison inside builds `"undefined-NaN"` and no dates match, so the calendar view stays empty despite completed challenges being recorded.

## Keyboard handler event shape mismatch

Input's `_onKeyDown` translates raw key events into semantic action names:

```javascript
this._emit('shortcut', { action: 'undo' });
this._emit('shortcut', { action: 'redo' });
this._emit('shortcut', { action: 'hint' });
this._emit('shortcut', { action: 'newGame' });
this._emit('shortcut', { action: 'autoComplete' });
this._emit('shortcut', { action: 'pause' });
this._emit('shortcut', { action: 'toggle' });
```

The v1 fix correctly changed the listener registration from `'keyboard'` to `'shortcut'`, but `_onKeyboard` still expects a raw `KeyboardEvent`:

```javascript
_onKeyboard(e) {
    const key = e.key;          // undefined
    const ctrl = e.ctrlKey || e.metaKey;  // undefined || undefined = undefined
    const shift = e.shiftKey;   // undefined

    if (ctrl && key === 'z' && !shift) { ... }  // never true
    ...
}
```

Since `key`, `ctrl`, and `shift` are all `undefined`, no condition ever matches. The fix:

```javascript
_onKeyboard(e) {
    if (!this.gameActive) return;
    switch (e.action) {
      case 'undo': this.game.undo(); this.usedUndoThisGame = true; this._positionCards(); break;
      case 'redo': if (this.game.redo) this.game.redo(); this._positionCards(); break;
      case 'hint': if (this.game.getNextHint) this.game.getNextHint(); break;
      case 'newGame': this._handleScreenAction('modeSelect'); break;
      case 'autoComplete': if (this.game.canAutoComplete && this.game.canAutoComplete()) this.game.startAutoComplete(); break;
      case 'pause':
        if (this.screens.isActive()) this._handleScreenAction('resume');
        else { this.game.state = GAME_STATES.PAUSED; this.screens.show('pause'); }
        break;
      case 'toggle': this.game.drawFromStock(); this._positionCards(); break;
    }
}
```

## Calendar always empty

`_syncScreenState` calls:

```javascript
calendar: this.dailyChallenge.getCalendarMonth ? this.dailyChallenge.getCalendarMonth() : [],
```

The method signature is `getCalendarMonth(year, month)`. Without arguments, `monthStr` becomes `"undefined-NaN"` and the `dateStr.startsWith(monthStr)` check never passes. The fix:

```javascript
const now = new Date();
calendar: this.dailyChallenge.getCalendarMonth(now.getFullYear(), now.getMonth()),
```
